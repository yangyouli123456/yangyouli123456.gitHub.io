| created | 没有el | 有 data

| mounted | 有 $el | 有 data

只执行一个生命周期： activated

因为这个页面或者组件已经被缓存了

  1. vue 系统自带的一个组件，功能：是用来缓存组件的，->提高性能

  2. 使用场景
    就是依赖缓存组件，提高项目的性能，具体实现比如：
    首页进入详情页，如果用户在首页每次点击都是相同的，
    那么详情页就没有必要请求n次了，直接缓存起来就可以了
    当然点击的不是同一个就直接请求。

    来获取dom的，比如jQuery的$()

    获取更新后的 dom 内容

  3. 作用 让样式在本组件生效

  4. 原理：给节点新增属性，然后css 根据属性选择添加的样式

  5. 在 样式上 就可以了

    父元素 /deep/ 子元素

    样式穿透
    父元素 >>> 子元素

    computed 和 methods 的区别?
    computed: 有缓存的
    methods ： 没有缓存
    watch：没有缓存

computed 和 watch 的区别？
watch 是监听，数据或者路由发生改变才可以响应()
computed计算某一个属性的改变，如果某一个值改变了，
计算属性会监听到镜像返回
watch：是当前监听到数据改变了，才会执行内部代码

props => methods => data => computed => watch

源码：
function initState(vm) {
var opts = vm.$options;
if (opts.props)
initProps$1(vm, opts.props);
// Composition API
initSetup(vm);
if (opts.methods)
initMethods(vm, opts.methods);
if (opts.data) {
initData(vm);
}
else {
var ob = observe((vm._data = {}));
ob && ob.vmCount++;
}
if (opts.computed)
initComputed$1(vm, opts.computed);
if (opts.watch && opts.watch !== nativeWatch) {
initWatch(vm, opts.watch);
}
}

vuex 单向数据流
state :类似于组件中的 data，存储数据
getters:类似于组件中的computed
mutation:类似于组件中的methods，同步
action:提交mutations的，任意异步操作
modules: 把以上 4 个属性再细分，让创库根号管理

显示： 通过path query 传值
this.$router.push({
path:’/about’,
query:{
a:1
}
})

接受：
created() {
console.log(this.$route.query.a);
},

隐式传值：

// 隐式 路由传值 在 路径上不会显示要传值的信息
this.$router.push({
name:’about’,
params:{
a:1
}
})

接受：
created() {
console.log(this.$route.params.a);
},

全局路由钩子：
beforeEach(to,from, next)、
beforeResolve(to,from, next)、
afterEach(to,from)；

独享路由钩子：
beforeEnter(to,from, next)；

组件内路由钩子：
beforeRouteEnter(to,from, next)、
beforeRouteUpdate(to,from, next)、
beforeRouteLeave(to,from, next)
